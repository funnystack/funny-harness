## 评审目标
重点检查以下内容：

- 是否偏离 OpenSpec 定义的范围
- 是否违反 Spring Boot 分层架构约束
- 是否把业务逻辑写进 Controller
- 是否绕过 Service 直接访问数据层
- 是否缺少测试
- 是否引入事务、SQL、分页、缓存相关风险
- 是否缺少文档或迁移说明
- 本次变更是否需要同步更新 AGENTS.md

## 忽略项
- 已由格式化工具处理的纯格式问题
- 没有实际价值的主观风格评论

## 必须指出的问题
- 超出本次 change 范围的实现
- 为了通过测试而削弱校验或异常处理
- 高风险 SQL 改动没有说明
- API 改动却没有测试覆盖
- 事务边界不清晰
- Mapper/Repository 调用方式存在明显风险
